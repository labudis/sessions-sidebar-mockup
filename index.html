A. Tabs

Pros

  • Keeps conversation view completely clean; sessions are only visible when you want them
  • Room for rich detail per session (branch, duration, logs) without space constraints
  • Familiar pattern; mirrors PR tabs (Conversation, Commits, Files changed)

Cons

  • Sessions are hidden by default; users may not notice an agent is actively working
  • Requires a full page switch, breaking reading flow
  • Adds a new top-level tab to an already crowded header on some issue types

B. Timeline Events

Pros

  • Sessions appear in chronological context alongside human comments and activity
  • Makes agent work feel like a natural part of the issue's story
  • No new UI surfaces to learn; uses existing timeline patterns

Cons

  • Can get buried in long conversations with many comments
  • Hard to get a quick overview of all sessions at a glance
  • Adds visual weight to the timeline, especially with multiple sessions

C. Pinned Banner

Pros

  • Immediately visible; running sessions are front and center when you open the issue
  • Collapsible, so users who don't need it can dismiss it
  • Keeps sidebar clean while still being prominent on the main content area

Cons

  • Pushes conversation content down, especially with several sessions
  • May feel intrusive on issues with no active sessions (if shown by default)
  • New UI pattern; no existing precedent in the issues page

D. Sidebar Section

Pros

  • Follows existing sidebar conventions (Assignees, Labels, Projects); minimal learning curve
  • Always visible without taking space from the main content area
  • Quick glanceable status; fits the "metadata at a glance" mental model of the sidebar

Cons

  • Sidebar is already crowded on issues with many fields (Projects, Milestones, Development)
  • Limited horizontal space constrains how much detail each session can show
  • Easy to scroll past on long sidebars; position relative to other sections matters

E. Header Pill

Pros

  • Always visible at the top; impossible to miss an active session
  • Minimal footprint; just a small pill until clicked
  • Dropdown provides full detail on demand without leaving the page

Cons

  • Header area is already dense; another element competes for attention
  • Dropdown overlays content, which can obscure important context
  • Not a standard pattern in the issues header; may feel unexpected
